# 可观测底座

> 「看得见」—— 稳定性治理（AIOps）的数据地基。本支柱沉淀 Soul 可观测能力的端到端建设：从事件中心、链路/日志/指标采集，到面向 GPU、Agent 的专项观测与容量评估。

## 包含

- **Soul-Bigeyes 事件中心** —— 事件模型、故障湖、产品层
- **soul-otel** —— OTel 链路与日志体系
- **metric 速通 / owl 报警配置中心** —— 指标与告警
- **APM agent 规划 / Agent 可观测（AgentOPS）** —— 应用与 Agent 观测
- **GPU 可观测** —— 利用率指标、排障、PAI 大盘
- **容量评估**
- **code wiki**

> 上层「稳定性治理 · AIOps」消费这里产出的事件 / 指标 / 日志数据做根因分析与自动化处置。
